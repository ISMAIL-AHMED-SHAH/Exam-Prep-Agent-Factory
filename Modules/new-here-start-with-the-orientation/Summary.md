## 1. Teen Bunyadi Terms

| Term | Matlab |
|--------|---------|
| **AI Worker / Digital FTE** | AI jo real job karta hai, sirf jawab nahin deta. "Full-Time Equivalent" — naya hire jo kabhi nahin sota, aap usay kaam dete hain, woh karta hai, aur human sign-off karta hai. |
| **General Agent** | All-purpose assistant (Claude Code, Cowork, ChatGPT). Aap isay apna kaam karwane ke liye ya AI Worker **banane** ke liye use karte hain. |
| **AI-Native Company** | Jab ek founder chhoti team aur bohat se AI Workers ke saath real business chalata hai, bade staff ke bajaye. |

> [!TIP]
> **Digital FTE = System**
>
> Digital FTE sirf model + prompt nahin hota.
>
> Yeh 4 cheezon ka combination hai:
>
> - Structured Specs
> - Domain Expertise
> - Engineering Architecture
> - Human Oversight

## 2. Five-Layer Cake of AI (Jensen Huang)

| Layer | Kya Hai |
|---------|----------|
| **1. Energy** | Data centers ko power dena |
| **2. Chips** | Specialized processors |
| **3. Infrastructure** | Cloud platforms aur supercomputers |
| **4. Models** | Neural networks jo seekhte hain |
| **5. Applications** | Jahan AI useful banta hai — yeh book isi layer par focus karti hai |

> [!IMPORTANT]
> Pehli 4 layers par billions of dollars invest kiye jate hain taa ke Layer 5 exist kar sake.
>
> Value sirf models se nahin aati — value tab create hoti hai jab intelligence workflows aur products mein transform ho jati hai.

## 3. Find Your Path — Teen Rungs Ki Ladder

| Rung | Kya Karte Hain |
|--------|----------------|
| **1. Foundations** | Short courses, browser mein, sab ke liye (doctor, accountant, student, engineer sab same) |
| **2. Mode 1** | AI se apna kaam tez karna — aap doer rehte hain, AI power tool hai |
| **3. Mode 2** | AI Workers banana jo aap ke liye kaam karein — aap builder ban jate hain |

> [!TIP]
> Poori ladder chadhna zaroori nahin.
>
> **Foundations + Mode 1** apne aap mein ek bohat strong aur practical skill set hai.

# 📗 Delivery Aur Audience

## 4. Four Channels, One Canonical Source

| Channel | Kya Hai |
|------------|----------|
| 📘 **The Book** | Canonical source — authoritative knowledge base |
| 💬 **TutorClaw** | Book khud sikha raha hai 24/7 (WhatsApp, Telegram, aur Web par) |
| 🛠️ **Skillpack** | Book Claude Code/OpenCode ke andar chal rahi hai |
| 📚 **Derivative Books** | Book har audience aur topic ke liye rewrite hui hai |

> [!NOTE]
> Jab source update hota hai, sab 4 channels update ho jate hain.
>
> **Methodology constant hai — channel variable hai.**

## 5. Two Tools, One Discipline

### 🔧 Claude Code

- Frontier-first
- Anthropic ke best models
- Complex multi-file refactors ke liye ideal

### 🔧 OpenCode

- Open-source
- Model-agnostic (Claude, GPT, Gemini, local models)
- Daily coursework aur flexibility ke liye ideal

> [!TIP]
> Skills, specs aur patterns portable hote hain.
>
> Agar aap `SKILL.md` Claude Code se OpenCode mein drop karein, to aam tor par woh waisi hi kaam karegi.

## 6. System of Record — Jensen Huang Ka Argument

Jensen Huang ka kehna hai:

> Agents system of record ki zaroorat khatam nahin karte — balke usay aur zyada zaroori bana dete hain.

Agents ko **ground truth** chahiye hota hai. Warna woh hallucinate kar sakte hain.

### Is Book Ka Role

| Component | Role |
|------------|--------|
| **Book** | Canonical source |
| **TutorClaw** | Teaching agent (sirf book se seekhta hai, open internet se nahin) |
| **Claude Code / OpenCode** | Building agents jo book ki methodology follow karte hain |
| **Human Judgment** | Verification layer (10-80-10 framework ka final 10%) |

> [!IMPORTANT]
> Education aur construction dono domains mein:
>
> **Book = System of Record**

## 7. Crash Courses vs Deep Dive Chapters

| Tier | Kya Hai |
|---------|----------|
| **Crash Courses** | Short, high-leverage primers jo daily-use ka ~80% cover karte hain aur aap ko hours mein productive bana dete hain |
| **Deep Dive Chapters** | Full comprehensive reference material jo real work mein gaps aane par consult kiya jata hai |

> [!TIP]
> Crash Courses speed ke liye hain.
>
> Deep Dives depth ke liye hain.

## 8. One-Person Billion-Dollar Company

### Sam Altman (January 2024)

Sam Altman ne predict kiya:

> "Ten-person billion-dollar companies aa rahi hain, aur mumkin hai pehli one-person billion-dollar company bhi dekhne ko mile."

### Expected Shape

| Component | Purpose |
|------------|----------|
| Canonical Source | Founder own karta hai |
| AI Agents | Woh kaam karte hain jo pehle human teams karti thi |
| Rented Infrastructure | Models, compute, harnesses aur providers |

> [!TIP]
> Agent Factory ecosystem khud isi pattern ki ek misaal hai:
>
> **Founder-Owned Knowledge + AI Workers + Rented Infrastructure**

---

## 🎯 Key Takeaway

AI ecosystem ko samajhne ke liye 3 concepts yaad rakhein:

1. **General Agents** → Aap ka kaam accelerate karte hain.
2. **Digital FTEs** → Aap ke liye kaam karte hain.
3. **AI-Native Companies** → Humans aur AI Workers mil kar businesses chalate hain.

Aur in sab ki foundation hai:

> **A reliable System of Record + Human Oversight**
