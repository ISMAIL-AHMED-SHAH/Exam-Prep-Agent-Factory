**Q3.** Among the 4 qualities of a good specification (Clarity, Completeness, Constraints, Testability), which is MOST directly responsible for preventing scope creep?

🅐 Clarity — clear language prevents misinterpretation, the primary cause of unplanned scope additions
🅑 Testability — measurable criteria create a fixed finish line that prevents continuous feature additions
🅒 Completeness — a complete spec leaves no room for additional features to be justified as 'covering missing cases'
🅓 Constraints — explicitly stating Non-Goals gives teams a documented boundary; without it every conversation becomes a scope negotiation

**Q4.** SDD's Phase 4 (Tasks) includes a specific task-sizing rule. What is it and why does it matter?

🅐 Each task must be completable by a single AI API call — multi-call tasks create state management complexity
🅑 Tasks must be written in imperative code-like syntax rather than natural language — AI parses structured commands more accurately
🅒 Each task must have exactly three acceptance criteria — no more, no less — for standardized automated quality checking
🅓 No task should exceed 2 hours — tasks that are too large create ambiguous scope that AI cannot execute precisely; smaller tasks allow AI to generate testable, reviewable increments

**Q6.** Mistake 4 in SDD errors is 'Treating Specs as Static.' What does the book prescribe instead?

**Q18.** Capability 2 warns that without 'observability,' agent deployments fail. What SPECIFICALLY happens?

🅐 Clients cannot see agent activity, requiring manual progress reports that undermine the scalability promise
🅑 You cannot charge outcome-based pricing without data to verify when and to what degree outcomes were achieved
🅒 Without feedback loops, agents cannot improve over time — each deployment stays at initial performance levels
🅓 Opaque systems behave unpredictably — when compliance breaches or quality lapses occur, you cannot diagnose why, defend decisions to regulators, or fix the problem; this erodes credibility and margin through costly remediation
